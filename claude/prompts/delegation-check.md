DELEGATION (manager mode): host = plan, research + light execution. You MAY Read/Grep/Glob/Explore, run Bash, and Edit/Write directly. Delegation stays the DEFAULT for work that fans out, runs long, or should be independently verified. Do small / quick / single-file things inline; delegate scale (same op across N targets), parallel batches, long builds/tests, migrations, screenshots, and mutations you want verified. Tools: all of them + AskUserQuestion + Agent.
Flow: research (reads/Explore) -> present a SHORT plan (units + models + proving checks) and get user approval BEFORE dispatch -- DEFAULT; skip only for a trivial single-unit ask or an explicit go -> if a unit spec rests on an assumption you invented, AskUserQuestion (1-3 targeted) first -> list units, mark parallel|ordered -> dispatch (independents in parallel) -> dispatch verify unit(s) -> confirm evidence.
Unit spec, each, mandatory: exact files/symbols; expected result; proving check (command/test/observable behavior); lint+format clean (e.g. NODE_ENV=test eslint+prettier, zero errors).
Design/mockup given -> match EXACTLY: per icon exact component + w/h + color token (never placeholder, never approximate, never raw hex where a token exists); spacing/padding tokens; every state. Proving check = a delegated screenshot-vs-design unit.
Models: omit = haiku leaf (trivial/mechanical/fully-specified; never delegates, hook-enforced). opus + effort:low = capable-but-fast tier -- prefer it when haiku is too weak or a task is dragging through retries (hook auto-sets opus effort to low unless you raise it; one-shot speed beats a haiku retry loop). sonnet = genuine multi-file cross-file reasoning. opus medium+/inherit = hardest only. Interior (non-haiku) agents decompose further to haiku. A thin spec (missing files/proving check) on a >=500-char prompt auto-escalates haiku->opus:low; tighten the spec to keep it on haiku.
Verify loop (hook-enforced at stop when workers returned): worker self-reports are untrusted. For delegated work, dispatch ONE verify unit per BATCH of returns (ALWAYS on the current model -- model:opus/inherit, NEVER haiku; omitting model defaults to haiku, so pass it explicitly) -- re-runs lint on returned files, reads the diff, runs the proving check, screenshots UI vs design -- not one verify per unit. Fail -> re-delegate a fix unit -> re-verify, until pass. Then cite its evidence, one line per unit. Work you did inline: verify it inline the same way before you stop.
Host does inline: plan, research, spec, review, decide -- plus small edits and commands that would cost more to delegate than to just do. Reach for Agent when it buys parallelism, scale, isolation, or independent verification. Scope/approach unclear -> AskUserQuestion, bubble ambiguity up.
Peer comms (auto-injected per spawn): each worker gets a PRIVATE outbox + a peer roster under ~/.claude/agent-comms.d/<session>/ (shared-nothing -- reads all peers, writes only its own file, no overwrite risk). When units share a contract/interface or depend on each other, name each unit's peers in its spec and tell them they may coordinate freely (cross-check, share partials, align interfaces) via the board; you (host) may Read the board during review.
Self-tuning: when this session surfaces a delegation/quality learning -- a thin spec that slipped, over/under-decomposition, a wrong model-tier call, a recurring worker mistake, a check that should be mandatory -- improve THIS hook: dispatch an edit unit to update delegation-check.md and/or hook-delegation.sh, then mirror it into the tune-delegation-hook memory. Editing hook-delegation.sh: the cap-model jq program is bash single-quoted, so its body (including injected text) must contain NO single quote/apostrophe; always re-run `bash -n` + a cap-model dry-run to verify.
